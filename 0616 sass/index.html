Sass

Sass 是 CSS的預處理器(preprocessor),目前在撰寫css時減少反覆重工的動作及更有效率的樣式結構維護，Sass 必須進行編譯(complier)成員生的CSS才能使用，也就是說瀏覽器不懂Sass，需要通過編譯才能閱讀。

CSS 不是程式語言，沒有流程控制、函式這種功能，Sass 除了可以使用嵌套規則(nested rules)、繼承(extends)、混合(mixins)這些功能，還可以使用變數(variables)、函式(Functions)、流程控制等等，所以也可以稱為 CSS 程式語言。

Sass 與 Scss

Sass 分為兩種， Sass 與 Scss，都稱為 Sass‧不同點如下:

副檔名 年份 語法 說明
.sass 2007年，最早的預處理器 縮排語法
沒有css大括號，結尾分號		 語法不能被大眾接受，無法兼容css
.scss 推出時期less之後 塊語法
寫法與css一樣，兼容css		 受到less影響，scss可以相容css，撰寫起來親和彈性，成為普及受歡迎的預處理器。

1.變數(Variables)

變數使用時機，當一個值被重複使用兩次以上時，就可以考慮使用

變數可以統一定義網站規範，只需改變數值，不須在CSS原始檔一一修改數值

底線(_)和連結線(-)的使用可以互換。定義變數名稱是$bg_black，但可以透過 $bg-black提取。

2. 嵌套(Nested)

嵌套雖然很好使用，不表示所以層級都該寫入嵌套裡面。可考旅是用最少的選擇起選擇到位置

過度崁入編譯後選擇器過長，會影響效能的執行，也會造成CSS檔案容量過大

以區塊嵌套有個優點，partials的概念讓結構區塊模組化，專案維護時只需針對錯誤找到模組位置修改

3. Extend(繼承)

繼承可以用css選擇器或%命名

使用CSS選擇器，會被編譯到CSS，而%佔位符不會被編譯到CSS

使用@extend盡量統一放在樣是最前面

相同的繼承樣式在編譯後會被使用群組選擇器合併在一起

4.計算

可做加、減、乘、除和取餘數的計算

除法比較特別，遇到處法的計算要用括號

Sass計算和calc()不同點在於預處理前已經計算號結果，不會造成瀏覽器負擔，calc()是在文件正在被柳覽器解讀時進行，會消耗瀏覽器資源

5. fuction函式

@function與@mixin有點像，都是在做運算處裡，兩者差異在於@function必須使用@retrun來返回結果，而且輸出的是一個值，並非一段CSS樣式，@mixin則可使用@inclide產生一段樣式

8.資料型態

  • 1. number(數字):數字有分無單位或有單位，1,2,13,10px
  • 2. string(字串):有(單)引號字串，"foo"、'範例'，無引號字串baz
  • 3. color(顏色):blue,#00000,rgba(0,0,0,0.5)
  • 4. boolean(布林值):true、false
  • 5. null(空值):null
  • 6. list(像陣列):用空格或逗號作為分隔。，1.5rem 1rem 0 2rem 或 "Helvetica,Arial,sans-serif"
  • 7. map(物件):類似js的物件

9. Partails (拆分之後在組織結構)

1.以底線來為partials檔案命名，Sass會自行判斷底線開頭的為partails檔案，有底線的partials檔不會產生css檔

2.載入partials方式為:@import 'variables' ，partials檔的底線和副檔名可以省略

3.許多partials檔，例如:mixin等等，會被用一隻主要的sass檔來載入它們
